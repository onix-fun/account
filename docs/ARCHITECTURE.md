# Архитектура

Сервис `account` спроектирован как единый "product image", который объединяет фронтенд, бэкенд и шлюз в один развертываемый модуль.

## Обзор компонентов

- **Frontend**: SPA на Vue 3, собранное с помощью Vite. Раздается Nginx как статика.
- **Backend**: Приложение на Ktor (Kotlin), предоставляющее REST и gRPC API.
- **Gateway (Nginx)**: Работает как обратный прокси, применяя строгий белый список публичных путей и раздавая фронтенд.
- **Менеджер процессов (s6-overlay)**: Управляет жизненным циклом Nginx и Ktor, обеспечивая поведение "fail-fast" (немедленная остановка контейнера при падении любого процесса).

## Сетевая топология

```text
Внешний трафик (Порт 8080)
      │
      ▼
   [Nginx] ───────────────────┐
      │                       │
      ├─ /api/* ──────────────┼──► [Ktor Backend (Порт 8081)]
      ├─ /assets/* ───────────┤             │
      └─ index.html ──────────┘             │
                                            │
                                            ▼
Внутренний трафик (Порт 9097) ───────► [gRPC Server]
```

## Проектирование безопасности

1. **Изоляция**: Nginx проксирует только определенные API-маршруты. Внутренние операционные эндпоинты (health, metrics) доступны только через внутренний порт (8081).
2. **Управление сессиями**: Безопасные куки с защитой от CSRF для веба и Bearer токены для API/gRPC.
3. **mTLS**: gRPC сервер требует обязательный взаимный TLS для безопасного взаимодействия между сервисами.
4. **Ограничение частоты (Rate Limiting)**: Интегрировано в бэкенд Ktor для предотвращения brute-force атак на чувствительные эндпоинты.
