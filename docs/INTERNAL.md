# Внутреннее устройство

## Бэкенд (Ktor)

Бэкенд написан на Kotlin с использованием фреймворка Ktor.

### Основные технологии
- **Koin**: Внедрение зависимостей (DI).
- **Exposed / PostgreSQL**: Доступ к базе данных и ORM.
- **Flyway**: Миграции базы данных.
- **Lettuce**: Клиент Redis для хранения сессий и лимитов.
- **Argon2**: Безопасное хеширование паролей.
- **Minio SDK**: S3-совместимое хранилище для аватаров.
- **gRPC Java**: Взаимодействие на базе Protobuf.

### Структура модулей
- `profile.auth`: Логика аутентификации, процесс регистрации и выпуск JWT.
- `profile.users`: Управление профилями и загрузка аватаров.
- `profile.sessions`: Отслеживание активных сессий и их отзыв.
- `profile.search`: Индексация и поиск пользователей на базе Meilisearch.
- `profile.infrastructure`: Клиенты для БД, Redis, SMTP и S3.

## Фронтенд (Vue 3)

Фронтенд представляет собой современное SPA, собранное на Vite.

### Основные технологии
- **Vue 3 (Script Setup)**: Composition API.
- **Pinia**: Управление состоянием (Auth, UI).
- **Vue Router**: Маршрутизация на стороне клиента с поддержкой подпутей.
- **PrimeVue**: Библиотека UI компонентов.
- **Axios**: HTTP-клиент с интерцепторами для CSRF и обновления сессий.

### Ключевые директории
- `src/features`: Бизнес-логика, разделенная по доменам (auth, profile, avatar).
- `src/api`: Автогенерируемые или ручные клиенты сервисов.
- `src/infra`: Настройки хранилища, навигации и интернационализации.
- `src/shared`: Переиспользуемые UI компоненты и утилиты.

## Управление жизненным циклом (s6-overlay)

Сервис использует `s6-overlay` для управления несколькими процессами.
- **`init-config`**: Скрипт-oneshot, запускающий `envsubst` при старте для генерации конфигураций рантайма.
- **`nginx`**: Долгоживущий процесс для раздачи веб-приложения.
- **`backend`**: Долгоживущий процесс Java для API.
- **Fail-Fast**: Если любой процесс завершается неожиданно, скрипт `finish` инициирует остановку всего контейнера.
