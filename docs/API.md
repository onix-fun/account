# API Контракты

## REST API (OpenAPI)

REST API документируется автоматически через Swagger UI.

- **Внутренний URL**: `http://localhost:8081/swagger-ui`
- **Публичный URL**: Обычно скрыт за Nginx (блокируется белым списком в продакшене).
- **Определение**: `http://localhost:8081/openapi.json`

### Публичные эндпоинты (Белый список)
Следующие пути доступны через публичный порт (`8080`):
- `/api/auth/*`: Регистрация, вход, выход, восстановление пароля.
- `/api/users/*`: Детали профиля, загрузка аватаров.
- `/api/sessions/*`: Управление сессиями.
- `/api/search/*`: Поиск пользователей.
- `/.well-known/jwks.json`: Публичные ключи для валидации JWT.

### Внутренние эндпоинты (Заблокированы)
Следующие пути доступны только на внутреннем порту (`8081`):
- `/livez`: Liveness probe (проверка работоспособности процесса).
- `/readyz`: Readiness probe (проверка готовности: БД, Redis, S3).
- `/metrics`: Метрики Prometheus.

## gRPC API

gRPC сервер обеспечивает высокопроизводительное взаимодействие между сервисами.

- **Порт**: `9097`
- **Путь к Protobuf**: `backend/src/main/proto/profile/v1/profile.proto`

### Основные сервисы
- `ProfileService`:
  - `GetProfile`: Получение данных пользователя по ID или имени.
  - `ValidateToken`: Проверка токенов внутренних сервисов.
  - `SearchUsers`: Массовый поиск и получение данных.

### Безопасность
- **mTLS**: Обязателен в продакшене. Сертификаты должны быть предоставлены через переменные окружения.
- **Client CA**: Разрешены только клиенты с сертификатами, подписанными доверенным CA.
